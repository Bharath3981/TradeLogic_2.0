# Screener v1 — Baseline

> **Status:** Active · First release
> **File:** `apps/api/src/services/screener/versions/v1.ts`

---

## Overview

v1 is the baseline screener. It fetches one year of daily candles for every FNO-listed stock and runs a composite technical analysis to produce a 0–100 score. The score drives the recommendation and the result ranking.

No custom weighting is applied — all indicators are treated equally by the underlying `analyzeStock()` utility. Future versions will tune these weights, add multi-timeframe confirmation, or introduce fundamentals.

---

## Universe

| Property | Value |
|---|---|
| Exchange | NSE (EQ segment only) |
| Symbol list | 56 FNO-eligible stocks |
| Sectors covered | Auto, Banking, Cement, Conglomerate, Consumer, Consumer Tech, Defence, Energy, Fintech, FMCG, Healthcare, Infrastructure, IT, Metals, Mining, NBFC, Pharma, Power, Retail, Telecom |

> Tokens are resolved dynamically via Kite's instruments API at scan time (not hardcoded).

## Timeframe & Lookback

| Property | Value |
|---|---|
| Candle interval | **Daily** |
| Lookback window | **1 year** (~252 trading sessions) |
| Minimum candles required | 50 (stocks with fewer are skipped) |
| Data source | Kite Connect Historical Data API |

## Indicators Used

### Trend & Momentum

| Indicator | Period | What it measures |
|---|---|---|
| **RSI** | 14 days | Overbought / oversold momentum |
| **MACD** | 12 / 26 / 9 | Trend direction + histogram momentum (expanding / contracting) |
| **EMA** | 20, 50, 200 days | Short-, medium-, and long-term trend alignment |
| **ADX** | 14 days | Trend strength (+DI vs −DI direction) |
| **Stochastic** | %K / %D | Short-term overbought / oversold |

### Volatility

| Indicator | Period | What it measures |
|---|---|---|
| **Bollinger Bands** | 20-day SMA ± 2σ | Price position relative to volatility envelope |
| **ATR** | 14 days | Average True Range as % of price (risk sizing proxy) |

### Volume

| Indicator | What it measures |
|---|---|
| **Volume ratio** | Current volume vs 20-day average (surge detection) |
| **Volume direction** | Bullish accumulation vs bearish distribution |

### Price Structure

| Indicator | What it measures |
|---|---|
| **52-Week High/Low** | Distance from annual extremes; flags breakouts |
| **Support & Resistance** | Swing highs/lows, EMA levels, round numbers — nearest S and R with touch-count strength |
| **Pivot Points** | Classic floor-trader pivots: PP, R1–R3, S1–S3 |
| **Candlestick Pattern** | Last-candle pattern recognition (doji, engulfing, hammer, shooting star, etc.) |
| **Trend Strength** | Combined direction (uptrend / downtrend / sideways) + strength (strong / moderate / weak) |

## Scoring

The composite score (0–100) is produced by `analyzeStock()` in `apps/api/src/utils/technicalIndicators.ts`. Each signal contributes equally — no indicator is weighted more than another in v1.

### Recommendation Thresholds

| Score range | Recommendation |
|---|---|
| ≥ 75 | **STRONG BUY** |
| 60 – 74 | **BUY** |
| 45 – 59 | **WATCH** |
| < 45 | **NEUTRAL** |

---

## Filtering (applied after scoring)

Results are filtered server-side before returning to the UI:

| Filter | Default | Description |
|---|---|---|
| Sector | ALL | Restrict scan to a single sector |
| Trend | ALL | Only return uptrend / downtrend / sideways stocks |
| Min Score | 40 | Discard stocks below this composite score |
| Holding Period | 3 months | Passed to `analyzeStock()` to tune signal sensitivity |
| Limit | Top 20 | Max results returned, sorted by score descending |

---

## Rate Limiting

Stocks are fetched in **batches of 3** with a **400 ms delay** between batches to stay within Kite Connect's historical data rate limits.

---

## Limitations (addressed in future versions)

- Equal weighting — no indicator is prioritised over another
- Single timeframe only (daily) — no weekly / monthly confirmation
- No fundamental filters (P/E, promoter holding, debt, etc.)
- Recommendation thresholds are static — not adaptive to market regime
- No backtesting validation of the score-to-return relationship

---

## Adding v2

To add a new version:

1. Create `apps/api/src/services/screener/versions/v2.ts` and export a `strategyV2` function with the same `ScoreStrategy` signature
2. Register it in `apps/api/src/services/screener/screener.registry.ts`
3. Create `docs/screener/v2.md` with equivalent documentation
4. Set the old version's `isLatest` to `false` and the new one's to `true`

No frontend changes needed — the version dropdown auto-populates from `GET /screener/versions`.
