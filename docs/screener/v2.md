# Screener v2 — 3-Layer Analysis

## Overview

Screener v2 introduces a **3-layer filtering pipeline** that combines fundamental quality, technical timing, and price action / derivatives confirmation into a single composite score.

| Layer | Name | Max Points | Data Source |
|---|---|---|---|
| L1 | Fundamental Analysis | 11 | Static quarterly data (`fundamentals.ts`) |
| L2 | Technical Analysis | 9 | Kite historical daily candles (1 year) |
| L3 | Price Action & Derivatives | 4 | Kite candles + NFO futures LTP |
| **Total** | **Composite** | **24** | |

**Normalized score**: `Math.round(rawTotal / 24 × 100)` → **0–100**

---

## Universe

~75 NSE F&O eligible stocks spanning:
Banking · IT · FMCG · Auto · Pharma · Metals · Energy · Power · Infrastructure · Defence · Insurance · Cement · Realty · Consumer Tech · Retail

Index filters (Nifty 50, 100, 500, etc.) intersect this universe via live CSVs from niftyindices.com.

---

## Layer 1 — Fundamental Analysis (0–11 points)

Each criterion is binary: 1 if condition is met, 0 if not.

| # | Criterion | Bullish Condition | Notes |
|---|---|---|---|
| 1 | Revenue Growth YoY | > 10% | Quarterly YoY |
| 2 | Net Profit Growth YoY | > 10% | Quarterly YoY |
| 3 | Return on Equity (ROE) | > 15% | Trailing 12M |
| 4 | Return on Capital Employed (ROCE) | > 12% | Trailing 12M |
| 5 | Debt-to-Equity | < 1 | Banks: auto-pass (D/E N/A) |
| 6 | Promoter Holding | > 40% | Banks: auto-pass (typically < 40%) |
| 7 | Promoter Pledging | = 0% | |
| 8 | PEG Ratio | < 1.5 | Skipped if PEG ≤ 0 |
| 9 | Interest Coverage Ratio | > 3× | Banks: auto-pass (999 = N/A marker) |
| 10 | Free Cash Flow | Positive | |
| 11 | EPS Growth 3Y CAGR | > 10% | |

**Bank / NBFC handling**: Debt-to-equity, promoter holding, and interest coverage ratios are not meaningful for banks. These criteria automatically pass (1 point each) for banking/NBFC stocks where `interestCoverage = 999`.

**Data refresh**: Update `apps/api/src/data/fundamentals.ts` quarterly after Q1/Q2/Q3/Q4 results season.

---

## Layer 2 — Technical Analysis (0–9 points)

Uses the same `analyzeStock()` utility as v1 (daily candles, 1-year lookback), but applies **stricter, v2-specific scoring rules** rather than v1's equal-weight approach.

| # | Indicator | Bullish Condition |
|---|---|---|
| 1 | 200 DMA | EMA signal = `bullish` (price above EMA 200) |
| 2 | 50 DMA / Golden Cross | EMA 50 > EMA 200 (Golden Cross) OR price above EMA 50 |
| 3 | RSI (14-period) | 40 ≤ RSI ≤ 65 — pulled back, not overbought |
| 4 | MACD Signal | `signal === 'bullish'` (crossover turning positive) |
| 5 | MACD Histogram | `histogram > 0` (positive momentum) |
| 6 | Volume | `direction === 'accumulation'` |
| 7 | ADX (14-period) | `adx.value > 20` AND `+DI > −DI` |
| 8 | Bollinger Bands | `signal === 'bullish'` AND `RSI > 30` |
| 9 | EMA 20 | EMA signal ≠ `bearish` (price holding above short-term MA) |

**Key difference from v1**: v1 gives equal weight across 13 indicators. v2 restricts RSI to the 40–65 sweet spot (not overbought), requires Golden Cross for the EMA-50 criterion, and explicitly requires +DI dominance for ADX.

---

## Layer 3 — Price Action & Derivatives (0–4 points)

| # | Signal | Condition |
|---|---|---|
| 1 | Bullish Candlestick | `candlePattern.signal === 'bullish'` |
| 2 | Support Respected | `supportResistance.signal === 'bullish'` (price at/near key support) |
| 3 | Volume Breakout | `volume.signal === 'surge'` AND `trendStrength.direction === 'uptrend'` |
| 4 | Futures Contango | Nearest futures LTP > equity LTP (positive premium %) from Kite NFO API |

**Futures data**: For each stock, the nearest expiry futures contract is looked up from the local instruments DB (`InstrumentRepository.findUpcomingFutures(symbol)`). LTP is fetched via `broker.getLTP()`. If no futures contract is found (e.g., instruments DB not synced for NFO), point 4 is skipped and the stock scores 0 for this criterion. The score still normalizes over 24.

## Trade Setup — Stop Loss & Target Calculation

Every screener result includes a computed **Trade Setup** object with a stop loss, two targets, and the technical basis for each level. The algorithm runs automatically after scoring (for both v1 and v2) via `calculateTradeSetup()` in `technicalIndicators.ts`.

---

### Stop Loss

The stop loss is determined by evaluating **6 technical methods in order** and picking whichever produces the **tightest (closest to current price)** technically-meaningful level. Tighter = less risk exposed per trade.

#### Method Priority (highest value wins)

| # | Method | Formula | Preferred when |
|---|---|---|---|
| 1 | **ATR-based** *(baseline)* | `price − (ATR₁₄ × atrMult)` | Always computed — guaranteed baseline |
| 2 | **Support Level** | `nearestSupport × 0.995` | Support exists and is above ATR-SL |
| 3 | **Recent Swing Low** | `highestSwingLow(last 20 bars) × 0.997` | Swing low is above ATR-SL |
| 4 | **EMA Floor** | EMA50 × 0.997 (uptrend) or EMA20 × 0.997 | Price is above the EMA and in an uptrend |
| 5 | **Bollinger Lower Band** | `BB.lower × 0.997` | Bullish candle pattern + price above BB lower |
| 6 | **Pattern Candle Low** | `lastCandle.low × 0.998` | Bullish reversal pattern detected (Hammer, Engulfing, Morning Star, Marubozu) |

> A method "wins" only if it produces a **higher** stop loss than the current best — meaning less downside risk.
> After all methods, the result is clamped: **SL must be 0.5% – 12% below the current price**.

#### ATR Multiplier by Holding Period

| Holding | `atrMult` | Effect |
|---|---|---|
| 1 month | 1.0× | Tight stop — short-term momentum trade |
| 2 months | 1.25× | Slightly wider |
| 3 months *(default)* | 1.5× | Standard swing trade buffer |
| 6 months | 2.0× | Wide stop — room for trend to breathe |

#### EMA Floor Logic (uptrend only)
- **Strong uptrend** (EMA signal = `bullish`): uses **EMA50** as the floor
- **Moderate uptrend** (EMA signal ≠ `bullish`): uses **EMA20** as the floor
- **6-month holds**: additionally checks **EMA200** as an absolute floor

#### Stop Loss Basis Label (shown in UI)
The UI displays which method was ultimately used, e.g.:
- `ATR×1.5` — pure ATR, no stronger level found
- `Support Level` — key support was closer to price
- `Recent Swing Low` — a local swing low was tighter
- `EMA50 Floor` — dynamic EMA support in confirmed uptrend
- `Hammer Low` — reversal candle's low is the natural invalidation point

---

### Target 1 (T1) — Conservative Target

T1 is the first, more achievable profit target. The algorithm selects the **lowest level that still gives a minimum 1.5:1 R:R**:

#### T1 Resolution (in order)

```
1. Nearest Resistance   → if (nearestResistance - price) / risk ≥ 1.5  → T1 = nearestResistance
2. Pivot R1             → if (R1 - price) / risk ≥ 1.5 AND R1 < T1    → T1 = Pivot R1  (tighter)
3. ATR Extension        → T1 = price + (risk × t1Mult)                  → fallback
```

> Step 2 can only **tighten** T1 (bring it closer), never push it out further.

#### T1 Multiplier by Holding Period

| Holding | `t1Mult` | Minimum R:R |
|---|---|---|
| 1 month | 1.5× | 1:1.5 |
| 2 months | 2.0× | 1:2.0 |
| 3 months *(default)* | 2.0× | 1:2.0 |
| 6 months | 2.5× | 1:2.5 |

#### Target Basis Label (shown in UI)
- `Resistance (2.3:1)` — nearest resistance level, R:R shown
- `Pivot R1 (1.8:1)` — pivot resistance is a tighter valid level
- `2.0:1 R:R` — no qualifying resistance, pure ATR extension

---

### Target 2 (T2) — Extended Target

T2 is the secondary, higher-conviction target for letting winners run.

#### T2 Resolution (in order)

```
1. Pivot R2   → if (R2 - price) / risk ≥ t2Mult × 0.75 AND R2 > T1  → T2 = Pivot R2
2. ATR Ext.   → T2 = price + (risk × t2Mult)                          → fallback
```

T2 is always clamped to be **≥ T1 × 1.01** (at least 1% above T1).

#### T2 Multiplier by Holding Period

| Holding | `t2Mult` | Typical gain |
|---|---|---|
| 1 month | 2.5× | ~5–8% move |
| 2 months | 3.0× | ~8–12% move |
| 3 months *(default)* | 3.0× | ~10–15% move |
| 6 months | 4.0× | ~15–25% move |

---

### Risk:Reward Ratio

```
riskRewardRatio = (T1 - currentPrice) / (currentPrice - SL)
```

The UI colour-codes this:
- **≥ 2.0** — green (good setup)
- **1.5–1.9** — amber (acceptable)
- **< 1.5** — red (unfavourable)

---

### Full Example (RELIANCE, 3M hold)

```
Current Price  : ₹2,950
ATR (14)       : ₹42

SL candidates:
  ATR×1.5      : ₹2,950 - ₹63   = ₹2,887
  Support Level: ₹2,900 × 0.995 = ₹2,885.5   ← skipped (below ATR SL)
  Swing Low    : ₹2,910 × 0.997 = ₹2,901.3   ← wins (tightest)
  → stopLoss = ₹2,901.30  (−1.65%)  basis: "Recent Swing Low"

Risk = ₹2,950 - ₹2,901.30 = ₹48.70

T1 candidates:
  Resistance   : ₹3,080  → R:R = (3080-2950)/48.70 = 2.67:1  ≥ 1.5  ← wins
  → target1 = ₹3,080  (+4.41%)  basis: "Resistance (2.7:1)"

T2:
  Pivot R2     : ₹3,180  → R:R = (3180-2950)/48.70 = 4.72:1  ≥ 2.25  ← wins
  → target2 = ₹3,180  (+7.80%)

riskRewardRatio = (3080 - 2950) / (2950 - 2901.30) = 2.67
```

```typescript
}
```

---

### Important Notes

- **Not a guarantee**: The stop loss and targets are derived from historical data and technical levels. They are reference points for trade planning, not absolute predictions.
- **Adjust for liquidity**: For stocks with wide bid-ask spreads, add a 0.1–0.2% buffer to the stop loss.
- **Futures lot size**: When using these levels for F&O trades, the actual rupee risk = `(currentPrice − stopLoss) × lotSize`.
- **Trailing stop**: Once T1 is hit, consider trailing the SL to break-even or below T1 to lock in profit.

---

## Rate Limiting

Same as v1: stocks processed in batches of 3 with a 400ms delay between batches to respect Kite's historical data API rate limits.

For v2, each stock in a batch also triggers one additional `getLTP()` call for its nearest futures contract. These are lightweight calls and are executed sequentially after the historical data fetch.

## Adding v3

1. Create `apps/api/src/services/screener/versions/v3.ts` implementing `ScoreStrategy`
2. Add entry to `SCREENER_REGISTRY` in `screener.registry.ts` with `isLatest: true`

4. Create `docs/screener/v3.md`
5. No frontend changes required — version dropdown auto-populates from the registry
